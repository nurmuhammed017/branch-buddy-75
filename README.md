# C2C Branch Delivery System

Веб-приложение для симуляции отказоустойчивой логистической системы доставки посылок между филиалами (branch-to-branch).

## О проекте

Система позволяет пользователям отправлять посылки из одного филиала в другой. Сотрудники филиалов обрабатывают и доставляют заказы, а администраторы контролируют весь процесс.

## Технологии

- **React 18** + **TypeScript** — UI и типизация
- **Tailwind CSS** — стилизация
- **Vite** — сборка и dev-сервер
- **LocalStorage** — персистентное хранилище
- **In-memory Cache** — быстрый доступ к данным

## Функциональные возможности

### Пользователь (User)
- Создание заказа: выбор филиала отправки и доставки, указание отправителя и получателя
- Просмотр своих заказов (как отправитель и как получатель)
- Визуальное разделение ролей: метки «Отправитель» / «Получатель»
- Отслеживание статуса заказа в реальном времени
- Просмотр таймлайна заказа с временными метками

### Сотрудник филиала (Branch Worker)
- Просмотр всех активных заказов
- Продвижение заказа по статусам: Created → Accepted → Shipped → Arrived → Stored → Delivered
- Валидация переходов (нельзя пропустить этап)

### Администратор (Admin)
- Полный обзор всех заказов в системе
- Просмотр журнала действий (логов)
- Мониторинг состояния системы

## Жизненный цикл заказа

```
Created → Accepted → Shipped → Arrived → Stored → Delivered
```

Каждый переход фиксируется с временной меткой. Пропуск этапов запрещён.

```
src/
├── components/       # UI-компоненты (LoginPage, UserPanel, WorkerPanel, AdminPanel)
```

## Триада CIA

- **Конфиденциальность** — пользователь видит только свои заказы
- **Целостность** — строгая валидация переходов статусов, защита от нелегальных операций
- **Доступность** — двухуровневое хранение: in-memory кеш + LocalStorage
